# RULE 7: No Silent Failures
After every tool call: IF failure -> REPORT exact error, RETRY once with correction, IF still failing -> ASK user.
DO NOT continue as if nothing happened.

---

# RULE 8: Critical Thinking - Devil Advocate Mode
Before implementing ANYTHING ask:
1. "Does this actually help users?" - if NO: push back, suggest alternatives
2. "Is there a simpler way?" - if YES: propose the shorter solution
3. "What breaks?" - identify edge cases, failure modes, race conditions

During: flag dead code, unclear naming, duplication, hardcoded secrets.
After: identify technical debt created, document with TODO + Linear issue.

---

Forbidden: code files, comments, documentation, CLI output, UI text, SESSION_SUMMARY.md body, commit messages.
Reason: UTF-8 encoding failures, tool compatibility, professionalism.

# RULE 16: Working Demos
2 working demos at each milestone (25/50/75/90/95%).
Each demo: runs without errors, demonstrates DIFFERENT aspect, max 3-step setup.
Types: CLI demo, screen recording, live URL, Jupyter notebook.
IF milestone reached with < 2 demos: STOP, create missing demos.

# RULE 17: Deep Understanding Before Phase Transition
Agent must ask user to:
1. Explain the mechanism: how does it work under the hood?
2. 2nd order: what happens at scale or in production?
3. 3rd order: what long-term dependencies or risks does this create?

Agent must: teach 1 new concept, ask all 5 critical thinking questions:
1. "What could break this in production we have not tested?"
2. "What happens if 10x users use this?"
3. "What assumptions are we making that might be wrong?"
4. "What would you do if this completely failed?"
5. "What surprised you most about building this?"

IF phase transition requested without this check: STOP, run checklist.
