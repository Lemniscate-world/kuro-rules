# RULE 56: v0.dev Landing Page Workflow
Trigger: L2 validation or when marketing needs a landing page.
Output file: docs/v0_landing_page_prompt.md

## Prompt Template for v0.dev
Build a clean landing page for [PROJECT NAME].

Hero section:
- Headline: [ONE sentence, core value, use Mom Test language]
- Sub-headline: [who it is for + what pain it solves]
- CTA: "Join waitlist" -> email form

Problem section (3 pain points from interviews):
1. [Pain 1]
2. [Pain 2]
3. [Pain 3]

Solution section: [Feature] -> [Benefit] (repeat 3x)
Social proof placeholder: "X people already waiting"
Footer: email capture, no tracking, no cookie banner
Style: Clean, professional, no gradients, no glassmorphism.
Stack: Static HTML + Tailwind only. No React. No framework.

## After export
1. Add form action: https://formsubmit.co/{your_email}
2. Add to .gitignore until launch
3. Deploy: npx -y surge {deploy_dir} {project_name}.surge.sh

# RULE 57: Parallel Validation Tracks
Trigger: at L2 validation.
Run ALL 3 tracks simultaneously (not sequentially).

## Track 1: Technical Validation (1 week)
Build smallest possible working demo.
Goal: prove core mechanic is technically feasible.
Output: demo URL or screen recording.

## Track 2: Market Validation (2 weeks)
Deploy landing page (R56). Drive 50-100 targeted visitors.
Goal: 10+ qualified email signups.
Output: signup list + conversion rate.

## Track 3: Distribution Validation (1 week)
Identify 3 communities. Post problem-framing post (NOT a product post) in each.
Goal: 3+ people respond with "I have this problem".
Output: documented community responses in `docs/tracking/acquisition_tracker.md`.

## Decision at End of Tracks
All 3 positive: FULL BUILD - proceed to R77 L2 pipeline
2/3 positive: CONDITIONAL BUILD - fix weak track first
1/3 positive: PIVOT - re-validate hypothesis
0/3 positive: STOP - document in KNOWLEDGE_BASE

## Enforcement
IF L2 validated: activate all 3 tracks simultaneously.
DO NOT run them sequentially (wastes 2-4 weeks).
DO NOT skip Track 3 because you "know" the community.
