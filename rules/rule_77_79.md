3. Community posts: Generate docs/community_post_template.md for Reddit, Discord, HN, X
4. Session Export: Run sync_summary.py -> .docx
5. Gitignore: Verify automate_l2.py and templates are in .gitignore

## Required Files
automate_l2.py | landing_page.html | docs/community_post_template.md | sync_summary.py

- Never ask user to create hosting account
- Never wait for response before deploying
- Static HTML only for landing page (no framework)

# RULE 79: SESSION_SUMMARY.md Compact Section

## Requirement
Every SESSION_SUMMARY.md MUST include "Résumé Compact" section.
**Audience**: Non-technical stakeholders, investors, and external partners. 
**Goal**: Explain progress WITHOUT technical jargon or code references.
Position: FIRST thing after ## Français / ## English headers.
Word limit: 150 words MAX per language. No exceptions.

## Content must answer
1. What problem was solved or progress made?
2. Why does it matter?
3. What is next?

## Rules
- No jargon without plain-language explanation
- No acronyms without explanation
- No code references
- Plain language: "L2 validation" -> "we confirmed the problem is real"

## Enforcement
- **Hard Lock**: No session is considered "CLOSED" without a verified compact summary.
- **Kuro Alert**: The Kuro daemon will flag any project missing this section as a "Critical Process Failure".
- Missing compact summary -> add before closing session.
- Over 150 words -> trim.
- Unexplained jargon -> rewrite in plain language.
