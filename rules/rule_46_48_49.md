# RULE 46: Web-Debug-7 Protocol
Trigger: any web/GUI bug, or before phase transition requiring 80% debug coverage.

## The 7 Steps (always in this order)
1. Console First: exact error message + stack trace
2. Network Inspection: HTTP status codes, response payloads, latency
3. State Audit: component state, props, context synchronized with backend?
4. Backend Correlation: cross-reference frontend errors with backend logs by timestamp
5. DOM/Style Verification: layout issues, hidden elements (z-index, display:none), ID collisions
6. Storage & Auth: LocalStorage, SessionStorage, Cookies - tokens valid?
7. Regression Suite: run full UI + E2E suite after any fix

## 80% Coverage Definition
- Auth flow: works end-to-end
- Primary user action: works without errors
- Data persistence: saves and retrieves correctly
- Zero unhandled console errors in primary flows

## Bug Fixed Standard
NEVER declare fixed without evidence from at least 3 of the 7 steps.
IF coverage < 80%: STOP feature work, continue debugging.

# RULE 48: Market Gravity Test (B2C Validation)
Trigger: Mom Test outreach fails due to unresponsive B2C users.

## Evidence Matrix (required)
| Signal Type | Required |
|-------------|----------|
| Price signal | 1+ competitor with paid tier |
| Volume signal | 10K+ users on related tool |
| Complaint signal | 3+ Reddit threads about the pain |
| Behavior signal | 1+ workaround tool built by users |

## Scorecard
Urgency | Budget signal | Integration load | Distribution - minimum passing score: 15/20

## Skin in the Game Metric (mandatory)
Either: 50+ qualified email signups OR $100+ in pre-orders/deposits

## Enforcement
IF Mom Test blocked: ask "Switch to Market Gravity Test?" IF YES: build Evidence Matrix + Scorecard first.
DO NOT write backend code until Skin in the Game metric met.

# RULE 49: Adversarial Mom Test Simulation
Trigger: real-world users unresponsive AND Market Gravity Test not enough.
Goal: INVALIDATE the idea, not confirm it.

## Setup (3 personas minimum)
Persona 1 - The Skeptic: expert in domain, "I have seen this before, it never works because..."
Persona 2 - The Satisfied User: current competitor user, "I already have X, why would I switch?"
Persona 3 - The Edge Case: power user, "This works for simple cases but breaks for mine because..."

## Rejection Mandate
At least ONE persona MUST reject with an UNRESOLVABLE reason.
If all 3 can be talked around: simulation is not strict enough. Redo harder.

## Pass Condition
1. Rejections addressed with concrete evidence (not optimism)
2. At least 2 of 3 personas end with "I would try this"
3. User confirms: "Simulation approved - adversarial enough"

## Enforcement
DO NOT run simulation where all personas agree.
DO NOT accept vague "yes" from simulated persona.
