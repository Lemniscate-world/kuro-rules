# RULE 5: Testing Requirements
## Minimum Coverage: 60% - no exceptions
| Type | Target | Scope |
|------|--------|-------|
| Unit Tests | 70% of test budget | Individual functions/methods |
| Integration Tests | 20% of test budget | Component interactions |
| E2E Tests | 10% of test budget | Complete user flows |

## Run Commands
Python: pytest --cov=. --cov-report=term-missing
Rust: cargo test + cargo tarpaulin
Node/React: npm test -- --coverage

## Enforcement
IF coverage < 60%: STOP new features, write tests until 60%+ reached

# RULE 6: Security Scanning
## Required before every commit
| Language | Commands |
|----------|----------|
| Python | bandit -r . && safety check |
| Rust | cargo audit && cargo clippy -- -D warnings |
| Node.js | npm audit --audit-level=high |

## Blockers: bandit HIGH, safety CRITICAL, cargo VULNERABILITY, npm HIGH/CRITICAL
## Enforcement: IF scan fails -> STOP commit, fix, re-run, DO NOT use --no-verify

# RULE 18: Regression Prevention
## After EVERY fix, feature, or refactor:
1. RUN full test suite (not just affected module)
2. CHECK: did previously passing tests now fail?
3. VERIFY: mocks match real production data structures
4. ENSURE: fake timers isolated and cleaned up per test
5. CONFIRM: no mock returns null when object/array expected

## Enforcement: IF regression detected -> STOP new work, fix immediately

# RULE 19: Strict Versioning (SemVer-Author)
## Format: v[Major].[Minor].[Patch]-kuro
## Tag at every validation milestone
git tag -a v0.1.0-kuro -m "25% milestone - core extraction working"
git push origin v0.1.0-kuro

## Enforcement: IF milestone reached without tag -> STOP, create tag immediately

# RULE 38: Mandatory Code Review After Commit
## Timing: AFTER commit -> review -> THEN continue
## Tools: CodeRabbit (preferred), Qode, GitHub PR, CodeClimate
## Review criteria: quality, security (R6), coverage (R5), regression risk (R18), docs

## Enforcement: IF no review after commit -> STOP, request review, wait for approval

# RULE 58: Frontend Coverage >= 90%
## Every view/page, user interaction, service/store method, and error state must be tested
## Tools: Vitest + @testing-library (React/Vue), pytest-qt (PyQt)
## Enforcement: IF frontend coverage < 90% -> STOP, write missing tests

---

# RULE 59: Integration & E2E Coverage >= 90%
## Minimum flows: happy path, error path, edge case path
## Tools: Playwright or Cypress (web), pytest with httpx (API)
## Enforcement: IF critical flows untested -> STOP before phase transition
