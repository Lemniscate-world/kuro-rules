# RULE 4: SESSION_SUMMARY.md Format

## Required format (every session, EN + FR)
# Session Summary - YYYY-MM-DD
**Editor**: (Cursor | VS Code | Windsurf | Antigravity)

## Resume compact / Compact Summary
[Max 150 words - see R79]

## Francais
**Ce qui a ete fait**: (liste)
**Initiatives donnees**: (nouvelles idees/directions)
**Fichiers modifies**: (liste)
**Etapes suivantes**: (ce qu il reste)
**Cry Test**: X%

## English
**What was done**: (list)
**Initiatives given**: (new ideas/directions)
**Files changed**: (list)
**Next steps**: (what next)
**Tests**: X passing
**Blockers**: (if any)
**Progress**: X% (pessimistic)
**Cry Test**: X%

## Rules
- NEVER overwrite history - always prepend new entry
- Both EN and FR required
- Progress % = pessimistic (R3 formula)
- Compact summary = first section (R79)
- Cry Test score mandatory (R71)

# RULE 42: Historical Log Preservation
## Rule: SESSION_SUMMARY.md MUST NEVER be overwritten. Prepend only.

## Process
1. READ existing content
2. PREPARE new entry
3. WRITE: new_entry + "\n---\n" + existing_content
4. VERIFY: old entries still present at bottom

## Enforcement
IF history deleted: STOP, restore from git (git show HEAD:SESSION_SUMMARY.md)
