# RULE 20: Hard Milestone Lock
Trigger: progress >= 25% AND "VALIDATION_PASSED - X%" NOT in SESSION_SUMMARY.md.
Lock means: NO new code, NO features, NO architecture changes.
Unlock: conduct validation (R14), write "VALIDATION_PASSED - 25%" in SESSION_SUMMARY.md.
Applies to CEO/Product persona only.

# RULE 21: Intelligence Harvester
Trigger: every milestone (10/25/50/75/90/95%). Required: 3 external sources minimum.
Sources: Reddit (sort Top/Year), App Store 1-3 star reviews, GitHub issues, HackerNews, G2/Producthunt.
Output: Intelligence Report saved in kuro-rules/MARKETING_MEMORY/<project>-intel-<date>.md
IF milestone reached without Intelligence Report: STOP, conduct research immediately.

# RULE 22: Feature Focus - One feature per validation cycle.

# RULE 23: Knowledge Capture
Trigger: any project pivot or termination.
Post-mortem format: What we built | Why we stopped | What failed (root cause) | Learnings | What we would do differently | Verdict
Save in: kuro-rules/KNOWLEDGE_BASE/<project>_postmortem.md
IF project pivots without post-mortem: STOP, document first.

# RULE 27: Persona Adaptability
Detect or ask persona at session start.
CEO: Why, business value, outcomes, timelines. Abstract tech (ASCII max). No deep code review.
DevOps/MLOps: How (infra), reproducibility, pipelines, containers, latency.
Developer: How (code), architecture, algorithms, DRY, SOLID.
Learner: Explain everything from scratch. Analogies before technical terms.
IF ambiguous: ask "On travaille en mode CEO ou Dev aujourd hui?"
