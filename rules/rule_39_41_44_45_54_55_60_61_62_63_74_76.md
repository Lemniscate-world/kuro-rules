# RULE 39: Pre-Marketing Pain-Point Due Diligence
Before any marketing, landing page, outreach, or product claim:
1. RUN Perplexity prompt (prompts/perplexity.md)
2. RUN Grok/X search (prompts/grok.md)
3. COLLECT 5+ recent signals, priority 2025-2026 sources
4. INCLUDE: 1+ official source, 1+ competitor signal, 1+ blog, 1+ social/forum
5. DOCUMENT: problem existence, urgency, compliance risks, what does NOT prove willingness to pay
6. VERDICT: desk research sufficient OR 3-5 expert calls needed?
IF research missing: STOP marketing claims, run templates first.

# RULE 41: Encoding Integrity - UTF-8 only, zero mojibake.
Common broken patterns: OCoO -> em dash | Ao (c) -> e accent | aooe -> smart quotes
Prevention: always write files with encoding='utf-8' in Python. Never use PowerShell > redirect.
Verification: python -c "open('AGENTS.md', encoding='utf-8').read(); print('OK')"
IF commit contains mojibake: REJECT, run fix_mojibake.py, re-verify.

# RULE 55: PR Ownership - Author Fixes Their Own Code
Reviewer: identifies issues, explains why, points direction, deposits feedback on Linear/GitHub PR, blocks merge.
NEVER fix the code for the author.
Exception: critical production emergency only - document it.
